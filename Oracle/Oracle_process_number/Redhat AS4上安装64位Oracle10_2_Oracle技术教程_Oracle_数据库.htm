Redhat AS4上安装64位Oracle10.2

 本文章是本人在Redhat AS4 OS 上安装oracle 10g DB时写的手稿,仅供大家参考.

    下面就开始介绍安装步骤，对于简单的部分就不多做说明了,不清楚的话可以网上查找：

   首先检查系统是否满足安装Oracle 10g的最低要求，其实这个对于目前大部分的服务器都是满足的，不过为了完整起见，还是把Oracle的最低要求和检查的语句列出来：

     1G以上内存；

     1G以上swap空间；

     /tmp目录下400M以上空间。

# grep MemTotal /proc/meminfo
# grep SwapTotal /proc/meminfo
# df -k /tmp
# grep "model name" /proc/cpuinfo

另外，需要保证Oracle的安装目录至少有超过2.5G以上的空间，才能保证企业版的正常安装。

通过下面的语句检查操作系统信息:

# more /etc/issue

# uname -a

如果上面的硬件条件满足了我们的需要,那么,我们还要看看系统本身的条件是否满足我们的需要,也就是操作系统是否已经包含了安装Oracle所需要的包,通过rpm -qa来检查以下包的版本,只有版本高于下面的时可以,如果低于此版本,则要升级处理,如下:

kernel-2.6.9-42.EL

glibc-devel-2.3.4-2.25

glibc-2.3.4-2.25

gcc-3.4.6-3

gcc-c++-3.4.6-3

compat-libgcc-296-2.96-132.7.2

compat-gcc-32-3.2.3-47.3

compat-gcc-32-c++-3.2.3-47.3

libstdc++-devel-3.4.6-3

libstdc++-3.4.6-3

make-3.80-6.EL4

gtk+-1.2.10-33

gnome-libs-1.4.1.2.90-44.1

tcp_wrappers-7.6-37.2

ORBit-0.5.17-14

libpng10-1.0.16-1

libungif-4.1.3-1.el4.2

imlib-1.9.13-23

setarch-1.6-1

libaio-0.3.105-2

audiofile-0.2.6-1.el4.1

esound-0.2.35-2

注意:  compat-libgcc-296-2.96-132.7.2这个包必须要加上，否则后面的安装无法继续进行.

当上面的安装包都达到我们的期望值后,再回过头来看看我们系统的核心参数,Oracle推荐的默认参数为：

将/etc/sysctl.conf文件中的核心参数进行修改,如下:

kernel.shmall = 2097152

kernel.shmmax = 2147483648

kernel.shmmni = 4096

kernel.sem = 250 32000 100 128

fs.file-max = 65536

net.ipv4.ip_local_port_range = 1024 65000

net.core.rmem_default=262144

net.core.rmem_max=262144

net.core.wmem_default=262144

net.core.wmem_max=262144

完成etc/sysctl.conf文件的修改后,重启OS.不过,通过/sbin/sysctl Cp可以修改当前核心参数，不过如果系统可以重启，还是通过这种方式比较保险。

修改/etc/security/limits.conf文件，将下面内容填入：(设置oracle用户的进程数和打开文件数的限制)

* soft nproc 2047

* hard nproc 16384

* soft nofile 1024

* hard nofile 65536

修改/etc/pam.d/login文件，添加如下的内容：

session required pam_limits.so  (注意要与此文件中的格式问题)

这些操作完成之后，开始安装的前期工作了，首先是要创建用户和组：

# groupadd oinstall

# groupadd dba

# useradd -g oinstall -G dba oracle

# passwd oracle

# mkdir -p /oracle/oracle/product/10.2   (在"/oracle"下建的)

# chown -R oracle.oinstall /oracle/oracle

当以上做完后,root用户的设置基本结束,现在可以切换到oracle用户了:

# su - oracle

$ vi .bash_profile       (添加环境变量)

  export LANG=zh.CN.UTF-8

  export ORACLE_BASE=/oracle/oracle     (先前建的)

  export ORACLE_HOME=/oracle/oracle/product/10.2(与前面对应)

  export ORACLE_SID=clo

  export NLS_LANG='american_america.ZHS16GBK'

        export PATH=$ORACLE_HOME/bin:$PATH

  export LD_ASSUME_KERNEL=2.6.22

然后,重启 .bash_profile,即可:

    $. .bash_profile(回车)

 注:ORACLE_BASE、ORACLE_HOME、ORACLE_SID以及NLS_LANG的设置都需要根据个人的具体进行调整.

接下来,我们进行软件的安装,安装的软件是:10201_database_linux_x86_64.cpio.

$cpio -idcmv < 10201_database_linux_x86_64.cpio

解压后,生成一个database的文件.

$cd database

$.runInstaller

此时,安装开始了,安装的过程中,将使用图形化来安装,不过要注意的时候,在安装过程中,我们还要执行两个 XXXXX.sh的脚本(脚本会在安装的过程中有有关的提示,但是在执行的时候要用oracle的权限去执行),同时,在进行图形化来安装的时候,我们必须将X导入到我们自己的本机,因为服务器是不让直接进行操作的,必须要进行远程操作,因此,客户端的本机也先前要安装有关X的软件,导入本机的方法是:

export DISPLAY=本客户机的IP:0.0

最后,安装完成了.有关建库和配置listener等的操作就非常的简单了.完了后,我们可以试着启一下数据库:

sqlplus "/as sysdba"

看信息,如果没有出现错误,就表示成功了,可以试着查查数据库中的表.
