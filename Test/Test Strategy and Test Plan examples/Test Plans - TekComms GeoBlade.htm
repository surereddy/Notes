QA Testing Timeline.

Testing Phases

pre-DC Testing Requirements

  • Developer supported integration tests 
  • Level 1 Tests complete

DC Milestone Requirements (before QA can start formal Integration Testing)

  • All JIRA development stories complete
  • All Major defects (Priority 1 and 2) Closed
  • System Integration fully complete (end to end)

DC-FR Testing Strategy

  • Regression coverage of existing 13.x functionality on the Iris Server
  • System Testing will be for the bulk of the testing cycle
  • KPI's, Robustness, and Performance testing will be required to be completed prior to declaring FR

Integration Test Start Requirements Validation (L1 Tests)

 

Once the below conditions have passed, we can declare Testing start for the first level of Integration testcases (pre-DC)

Hardware 

  • QA XCC1000 availability
  • Minimum 2 of each blade type available

PLAT / Base probe

  • Software Builds - regular AUTH builds in Tekplat-PE??  Or should we use TekPlat?
  • Software Builds - build name matching Jenkins build output (defect tracking purposes in Jira)
  • Complete New Probe installation (through probeSetup)
  • Applications start/restart without workarounds
  • Probe upgrade plat and applications without workarounds (apps and blades restart without intervention after upgrade)
  • Plist management - clarity on plist modification requirements
  • Blade basic communication
    1. Board Bringup (IPMC level/ping test)
    2. SOL Functionality
  • IFC Validation
    1. Board bringup (ilogin from MBC)
    2. Initial Core Diag test results
    3. Simple statistics test (packet counting from external port ingress)
  • IFP Validation
    1. Board Bringup (ilogin from MBC)
    2. Initial Core Diag test results
    3. Simple statistics test (packet counting from external IFC ingress)
  • IAP Validation
    1. Board Bringup (ilogin from MBC)
    2. Initial Core Diag test results
    3. Packet processing test (receive packets sent through PRE and IFC ingress)

Storage

  • SSU Connectivity to probe
  • Storage bringup - login from probe without workarounds
  • Packet storage verification

OAM

  • Software Builds - regular AUTH builds in Server-PE
  • Software Builds - buildname matching Jenkins build output (defect tracking purposes in Jira)
  • G10 probe basic regression tests (storage profile mtce, campaigns, ISA trace)
  • Iris server installation and upgrades

Integration

  • Probe connectivity to Iris Server
  • Maintenance of probe using Iris Server 
  • ISA Validation with Geoblade
  • IPA Validation with Geoblade

 

Design Complete - System Test Start Requirements Validation

 

Once the below conditions have passed, System Test can officially begin

Design Complete

  • No open JIRA Stories
  • No open Priority 1 or 2 defects
  • All design testing complete

Integration

  • Realtime traffic capture (no storage)
  • Historical traffic capture (with storage)

  • No labels
  • Edit Labels
